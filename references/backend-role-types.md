# 后端岗位类型参考

## 初级 Java 后端

重点判断：

- Java 基础、集合、异常、IO、多线程基础。
- Spring Boot、MyBatis、MySQL 基本使用。
- 是否能清楚讲项目流程、接口设计、表结构、异常处理。
- 是否有良好工程习惯：日志、参数校验、单测、Git、接口文档。

简历优化重点：

- 不要堆技术名词，突出自己负责的接口、模块、表设计和问题解决。
- 项目指标可以是数据量、接口耗时、缺陷下降、开发效率提升。

高频追问：

- HashMap 为什么线程不安全。
- MySQL 索引为什么会失效。
- Spring Bean 生命周期。
- 事务失效场景。
- 项目里最复杂的接口怎么设计。

## 中级 Java 后端

重点判断：

- 能否独立负责业务模块。
- 是否理解缓存、MQ、事务、并发、性能优化。
- 是否处理过线上问题。
- 是否能说明技术选型和取舍。

简历优化重点：

- 每个项目至少准备一个技术难点和一个线上问题闭环。
- 增加 QPS、RT、数据量、错误率、慢 SQL、消息量等指标。

高频追问：

- 缓存与数据库一致性。
- MQ 如何保证可靠投递和幂等消费。
- 分布式锁的正确使用。
- 慢 SQL 如何排查。
- 线上接口突然变慢如何定位。

## 高级 Java 后端 / 技术负责人

重点判断：

- 架构设计、系统稳定性、复杂问题拆解。
- 跨团队协作、技术规划、工程规范。
- 高并发、高可用、可观测性、成本意识。
- 能否讲清楚演进过程和技术债治理。

简历优化重点：

- 使用“业务目标 -> 架构方案 -> 关键取舍 -> 工程保障 -> 结果指标”表达。
- 突出 owner 意识，不只写开发实现。

高频追问：

- 订单/支付/库存系统如何保证一致性。
- 秒杀系统如何限流、削峰、扣库存。
- 服务不可用时如何降级和恢复。
- 如何设计监控指标和告警。
- 如何推动技术改造落地。

## 微服务后端

重点判断：

- 服务拆分边界、注册发现、配置管理、网关、鉴权。
- 限流、熔断、降级、重试、链路追踪。
- 分布式事务和最终一致性。

高频追问：

- 服务拆太细会有什么问题。
- Feign/RPC 调用超时如何治理。
- Sentinel/Hystrix 的核心思想。
- 分布式事务如何选型。
- 链路追踪如何定位慢调用。

## 高并发 / 中间件方向

重点判断：

- 缓存、队列、线程池、连接池、批处理、异步化。
- 压测、容量评估、热点隔离、降级预案。
- 对底层原理有足够理解。

高频追问：

- 线程池参数怎么设置。
- Redis 热 key 和大 key 怎么处理。
- MQ 堆积如何排查。
- 数据库连接池打满怎么办。
- 如何做容量评估和压测。
